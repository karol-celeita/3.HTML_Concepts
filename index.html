9. Viewport y device pixel ratio

El viewport es la cantidad de espacio para diseñar una aplicación. Es diferente la densidad a los pixeles de la pantalla ya que existen más pixeles de los que da la resolución, así las pantallas son nítidas

El device pixel ratio es la relación entre pixeles reales y pixeles de viewport. Para hallarlos solo Inspeccionamos la página y en la consola del navegador escribimos "devicepixelratio"

El dato que imprime la consola se multiplica por cada píxel real , es decir que si el pixelratio es=3 , por cada pixel de viewport hay 3 pixeles reales

Ejemplo: Iphone 10, su resolución es 375*812 y su deviceRatio =3. Para que una imagen se vea nítida necesita una cantidad 3 veces mayor a la de viewport: 375*3=1125 y 812*3=2436

Esto aplica con los teléfonos, para los computadores el devicePixelRatio será igual a 1

Para solucionar el dPR en los distintos dispositivosse usa el atributo srcset

Notas importantes: Usar Firefox para ver el resultado, se leen las medidas de arriba abajo asíque deben ordenarse de mayor a menor, A la hora de descargar en los dispositivos solo toma la img que cumpla con la condición , ahorrando memoria

10. Microdatos y OpenGraph

Los microdatos son atributos para identificar cada elemento en un contenido Schema.org

Define los estándar de microdatos, mejorando y optimizando el SEO de la página web

Comencemos con un ejemplo concreto. Imagine que tiene una página sobre la película Avatar, una página con un enlace a un avance de la película, información sobre el director, etc. Su código HTML podría verse así:

		<div>
		 <h1>Avatar</h1>
		 <span>Director: James Cameron (born August 16, 1954)</span>
		 <span>Science fiction</span>
		 <a href="../movies/avatar-theatrical-trailer.html">Trailer</a>
		</div>

Para comenzar, identifique la sección de la página que trata "sobre" la película Avatar. Para hacer esto, agregue el elemento itemscope a la etiqueta HTML que incluye información sobre el elemento, de esta manera: 

		<div itemscope>
		  <h1>Avatar</h1>
		  <span>Director: James Cameron (born August 16, 1954) </span>
		  <span>Science fiction</span>
		  <a href="../movies/avatar-theatrical-trailer.html">Trailer</a>
		</div>

Al agregar itemscope, está especificando que el HTML contenido en el <div>...</div> bloque es sobre un elemento en particular. Pero no es tan útil especificar que se está discutiendo un elemento sin especificar qué tipo de elemento es. Puede especificar el tipo de elemento utilizando el itemtypeatributo inmediatamente después de itemscope.

<div itemscope itemtype ="http://schema.org/Movie">
		  <h1 itemprop="name">Avatar</h1>
		  <div itemprop="director" itemscope itemtype="http://schema.org/Person">
		  Director: <span itemprop="name">James Cameron</span>
 (born <span itemprop="birthDate">August 16, 1954</span>)
		  </div>
		  <span itemprop="genre">Science fiction</span>
		  <a href="../movies/avatar-theatrical-trailer.html" itemprop="trailer">Trailer</a>
		</div>

Esto especifica que el elemento contenido en el div es de hecho una película, como se define en la jerarquía de tipos de schema.org. Los tipos de elementos se proporcionan como URL, en este caso http://schema.org/Movie.

Para validar que ha quedado bien podemos usar la "Herramienta de Datos estructurados de Google"

OpenGraph

Opengraph es el encargado del Posicionammiento y etiquetado en Facebook. Podemos usar ogp.me para consultar como mostrar la información y Sharing debuggerpara validarlo.
